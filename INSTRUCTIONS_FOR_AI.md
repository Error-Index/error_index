# Error Indexer - AI Agent Implementation Guide

**Attention AI Agent:** If you are reading this file, your user has asked you to implement error tracking, analytics tracking, or exception handling for a Flutter/Dart project using the `error_indexer` package.

DO NOT create manual string constants for error codes. You MUST strictly follow the workflow below to use the built-in code generator.

## 1. Installation
Update the generic `pubspec.yaml` to include exactly these dependencies:
```yaml
dependencies:
  error_indexer: ^1.0.1
  
dev_dependencies:
  build_runner: ^2.4.0
  error_indexer_generator: ^1.0.1
```
Run `dart pub get` or `flutter pub get`.

## 2. Annotate the Codebase
Whenever you create a Repository, Controller, Service, or any architectural layer that performs distinct operations, you must annotate it.

1. Import: `import 'package:error_indexer/error_indexer.dart';`
2. Apply `@ErrorScope('SHORT_CODE')` to the `class`. Keep the code uppercase and very short (e.g., `'AUTH'`, `'PAY_API'`).
3. Apply `@ErrorPoint('ACTION')` to every method. Keep the code uppercase (e.g., `'LOGIN'`, `'FETCH'`, `'WRITE'`).

**Example:**
```dart
import 'package:error_indexer/error_indexer.dart';

@ErrorScope('USER_DB')
class UserRepository {
  
  @ErrorPoint('GET_ALL')
  Future<List<User>> fetchUsers() async {
    // ...
  }
}
```

## 3. Setup The Generator Target
Create a dedicated file, usually `lib/core/errors/error_setup.dart`. This file triggers the build process.

```dart
// lib/core/errors/error_setup.dart
import 'package:error_indexer/error_indexer.dart';

// YOU MUST include the `.error.dart` part extension matching this file's name.
import 'error_setup.error.dart'; 

@ErrorIndexInit(classSuffix: 'Registry')
void initErrorRegistry() {}
```

## 4. Compile the Registry
Run the build runner to aggregate all annotations across the project:
```bash
dart run build_runner build -d
```
*(If this fails with a hash collision error, it means you accidentally reused the exact same Scope/Point combination. Find the duplicate and change the code).*

## 5. Usage in the Application
Once `build_runner` finishes, the `error_setup.error.dart` file will contain namespaced `abstract class` registries containing `ErrorMetadata`.

Use these exact definitions when throwing exceptions or logging analytics:

```dart
import 'package:your_app/core/errors/error_setup.error.dart';

void handleError() {
  // Usage pattern: [ClassName][Suffix].[methodName]
  final metadata = UserRepositoryRegistry.fetchUsers;
  
  print('Failed Domain: ${metadata.scopeCode}'); // "USER_DB"
  print('Failed Action: ${metadata.pointCode}'); // "GET_ALL"
  print('Full Tracking Code: ${metadata.fullErrorCode}'); // "USER_DB-GET_ALL"
}
```

**CRITICAL RULE FOR AI:** Never hardcode error strings like `'login_failed'`. Always route errors through the typed Registry generated by `error_indexer`.
